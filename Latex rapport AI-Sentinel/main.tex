\subsection{Architecture du Projet ``AI-Sentinel'' : Approche Modulaire et Multilangage}
Pour répondre aux exigences de performance et d'accessibilité de notre système de détection d'intrusions (IDS), nous avons opté pour une architecture strictement modulaire. Le projet est divisé en trois programmes distincts qui communiquent entre eux. Cette séparation des tâches permet d'utiliser les forces spécifiques de chaque langage (C et Python) tout en rendant le code plus facile à maintenir.

\subsubsection{Le Capteur Réseau : Sentinel-Sensor (Code en C)}
\begin{itemize}
    \item \textbf{Rôle :} C'est le ``bouclier'' et les ``yeux'' de notre programme. Il s'occupe de la capture des données brutes du réseau Wi-Fi.
    \item \textbf{Pourquoi le langage C ?} Le C a été imposé et choisi pour sa proximité avec le matériel (bas niveau) et sa gestion stricte de la mémoire. Pour écouter un réseau en temps réel, il faut un langage extrêmement rapide qui ne sature pas le processeur.
    \item \textbf{Fonctionnement concret :} Ce programme intercepte les trames réseaux (IP source, ports, taille du payload, contenu) et les formate instantanément sous forme de texte dans un fichier de journalisation continu (le fichier \texttt{traffic\_logs.csv}). Il agit comme un producteur de données.
\end{itemize}

\subsubsection{Le Moteur de Détection IA : Sentinel-AI (Code en Python)}
\begin{itemize}
    \item \textbf{Rôle :} C'est le ``cerveau'' du projet. Il dispose d'une logique algorithmique pour détecter les anomalies et les attaques.
    \item \textbf{Pourquoi le langage Python ?} Python excelle dans le traitement de données et l'implémentation de logique algorithmique complexe. Il est idéal pour calculer des statistiques en temps réel (comme les Z-scores sur la taille des paquets) et manipuler des chaînes de caractères pour détecter des signatures d'attaques.
    \item \textbf{Fonctionnement concret :} Ce script lit le fichier CSV généré par le programme C en ``Live Mode''. Il applique des filtres de sécurité pour identifier les comportements malveillants :
    \begin{itemize}
        \item \textbf{Flood / Scan de ports :} Détectés par une fréquence anormale de requêtes.
        \item \textbf{Brute Force / Injections (ex: SQL) :} Détectés par l'analyse du contenu du payload.
    \end{itemize}
    \item \textbf{Score de risque :} Le programme calcule un score final et affiche une interface console dynamique et colorée alertant l'utilisateur sur la gravité de la menace.
\end{itemize}

\subsubsection*{L'Interface Utilisateur : Sentinel-Launcher (Orchestrateur Python)}
\begin{itemize}
    \item \textbf{Rôle :} Rendre l'outil accessible à un utilisateur novice (``lambda'') via une interface graphique simple (bouton Activer/Désactiver).
    \item \textbf{Le ``Pourquoi'' de cette approche :} En cybersécurité, un outil n'est efficace que s'il est utilisé. Demander à un utilisateur sans compétences informatiques d'ouvrir un terminal, de compiler du C, puis d'exécuter un script Python avec des arguments est inenvisageable. Nous avons donc décidé de masquer toute la complexité technique derrière une application ``Plug \& Play''. L'utilisateur clique sur un bouton, et le système se protège tout seul.
    \item \textbf{Le ``Comment'' (Implémentation technique) :} Nous avons créé un script Python supplémentaire qui agit comme un chef d'orchestre.
    \begin{itemize}
        \item \textbf{L'interface visuelle :} Elle est générée grâce à la bibliothèque native \texttt{tkinter}, qui permet de dessiner une fenêtre applicative (boutons, labels d'état) sans avoir besoin d'installer de modules externes complexes.
        \item \textbf{L'orchestration (Le module \texttt{subprocess}) :} Lorsque l'utilisateur clique sur ``Activer'', ce programme utilise la bibliothèque \texttt{subprocess}. Cette bibliothèque permet de lancer d'autres exécutables en arrière-plan (processus enfants). L'orchestrateur va donc lancer silencieusement notre Sentinel-Sensor (C) en tâche de fond, puis ouvrir la console de notre Sentinel-AI (Python). Ainsi, les deux programmes s'exécutent simultanément et communiquent de manière totalement transparente pour l'utilisateur.
    \end{itemize}

\end{itemize}

\section{Difficultés rencontrées}
La principale difficulté a été  d’assurer une communication fiable entre le module en
C et le programme Python. Il a fallu définir un format de données clair afin d’éviter
les erreurs lors du traitement des fichiers CSV. Nous avons aussi dû gérer les lignes
incomplètes, les caractères inattendus et les journaux suspects. Les tests ont demandé
du temps, car plusieurs cas devaient être simulés pour vérifier la fiabilité du système.
Ces difficultés nous ont permis de mieux comprendre l’importance d’une architecture
simple, robuste et sécurisée.

\subsection{Conclusion}
Le projet atteint ses objectifs en proposant un système capable d’analyser des journaux et de repérer des anomalies. L’utilisation du C permet un filtrage rapide, tandis
que Python facilite l’analyse et la détection intelligente. Cette organisation rend le projet plus clair, plus fiable et plus simple à faire évoluer. Même si certaines améliorations restent possibles, comme l’ajout d’alertes automatiques, la base du système est fonctionnelle. Ce travail nous a permis de mettre en pratique nos compétences en programmation et en cybersécurité.

les alertes en temps réel sur un tableau de bord.

\section{Manuel utilisateur}
Le système complet de surveillance peut être exécuté selon votre système d'exploitation (Windows ou Mac/Linux).

\subsection*{1. Version Windows}

\subsubsection*{Option 1 : Via l'invite de commandes}
Ouvrez votre terminal Windows (Invite de commandes ou PowerShell) dans le dossier du projet et lancez la commande suivante :
\begin{verbatim}
python sentinel_main.py trafic_reseau.csv
\end{verbatim}

\subsubsection*{Option 2 : Lancement rapide ("Zéro Manipulation")}
Vous pouvez également double-cliquer directement sur le script d'exécution \texttt{Démarrer\_Sentinel.bat} créé à la racine du projet. Ce script contient les lignes suivantes :
\begin{verbatim}
@echo off
python sentinel_main.py trafic_reseau.csv
\end{verbatim}

---

\subsection*{2. Version Mac / Linux}

\subsubsection*{Option 1 : Via le Terminal}
Ouvrez votre Terminal dans le dossier du projet et lancez la commande suivante :
\begin{verbatim}
python3 sentinel_main.py trafic_reseau.csv
\end{verbatim}

\subsubsection*{Option 2 : Lancement rapide}
Vous pouvez exécuter le script Shell nommé \texttt{start.sh} créé à la racine du projet. Ce script contient les lignes suivantes :
\begin{verbatim}
#!/bin/bash
python3 sentinel_main.py trafic_reseau.csv
\end{verbatim}
Pour rendre ce script exécutable la première fois, lancez la commande \texttt{chmod +x start.sh} dans votre terminal.
